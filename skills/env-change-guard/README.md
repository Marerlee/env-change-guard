# env-change-guard

一个更适合新手的 OpenClaw 技能。

## 这个技能做什么

这个技能现在主要做三件事：

1. 记录环境变量或系统变量改了什么
2. 用电脑小白能听懂的话解释这次改动会有什么影响
3. 当用户看不懂错误或日志时，先查 log，再继续排查其他可能问题

它不追求复杂流程，也不想写成很重的技术规范，重点就是：

- 改了什么，要记下来
- 改完会影响什么，要讲清楚
- 用户看不懂报错时，先帮他看 log，再继续判断别的问题

## 适合什么场景

- 给 Python / Node / Java 加 PATH
- 设置 `JAVA_HOME`
- 配置 ADB / Android SDK
- 修改系统变量
- 用户明确表示自己不懂电脑配置，希望解释简单一点
- 用户遇到看不懂的错误、日志、命令报错

## 默认记录位置

`logs/env-change-log.md`

## 文件结构

- `SKILL.md` —— 技能主说明
- `README.md` —— 给人看的介绍

## 说明

这是一个极简版技能。

如果你希望代理在环境变量相关场景下：
- 留下变更记录
- 顺便给新手解释影响
- 看不懂报错时先查 log

那它就够用了。
