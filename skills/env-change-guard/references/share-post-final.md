# env-change-guard 对外发布文案（正式版）

## 版本一：适合发群 / 发朋友圈 / 发社区

我做了一个给 OpenClaw 新手用的小技能：`env-change-guard`。

它专门处理环境变量、系统变量、PATH 这类很容易把人搞晕的配置操作。

这个技能的思路不是“帮你猛改系统”，而是尽量把这件事做稳一点：

- 修改前先解释要做什么
- 先读取当前值
- 判断这次操作风险高不高
- 尽量只做最小改动
- 改完自动记日志
- 告诉你怎么验证
- 告诉你怎么回滚

默认日志会写到：
`logs/env-change-log.md`

适合这些场景：
- 配 Python / Node / Java / ADB / Android SDK
- 给 PATH 加目录
- 设置 `JAVA_HOME`
- 排查命令找不到
- 清理混乱的 PATH

如果你属于“知道要改环境变量，但又怕把电脑搞乱”的那类人，这个技能应该会挺有用。

## 版本二：适合发帖子 / 项目介绍

很多人在用 OpenClaw 时，都会碰到一类很容易出问题的操作：改环境变量。

比如：
- 给 Python、Node、Java 加 PATH
- 设置 `JAVA_HOME`
- 配 Android SDK / ADB
- 排查为什么命令找不到
- 清理已经越来越乱的 PATH

这类操作看起来不大，但对新手很不友好。最常见的问题就是：
- 不知道自己改了什么
- 改完为什么不生效也不清楚
- PATH 被覆盖了
- 出问题以后不会回退

所以我做了一个小技能：`env-change-guard`。

它不是一个“自动乱改系统”的技能，而是一个“环境变量防呆技能”。

它会要求代理在每次修改环境变量前都先做几件事：
- 先说明要改什么
- 先读取当前值
- 判断风险等级
- 尽量做最小改动
- 改完写日志
- 告诉用户怎么验证
- 告诉用户怎么回滚

默认日志位置：
`logs/env-change-log.md`

这个技能特别适合：
- OpenClaw 新手
- 不太懂系统变量 / 环境变量的用户
- 经常帮别人配开发环境的人
- 想让配置过程更可追溯、更不容易失控的人

如果你也踩过 PATH 的坑，应该会明白这种“改完有记录、出错能回滚”的安心感有多重要。
