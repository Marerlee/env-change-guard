# Windows 使用说明

## 作用范围说明

- 进程级：只影响当前终端或当前程序
- 用户级：影响当前 Windows 用户后续启动的新进程
- 系统级：影响整台机器上的所有用户，通常需要管理员权限

## 常见坑

- 把 `Path` 和临时变量混在一起
- 直接覆盖 `Path`，而不是只增加一段目录
- 改完以后忘了重开终端，误以为没生效
- 加了重复路径，导致实际调用的是旧版本工具
- 32 位和 64 位工具目录混用

## 面向新手的解释方式

建议按这个顺序解释：

1. 这个变量是干什么的
2. 哪些程序会用到它
3. 这次改动影响当前用户还是整台机器
4. 出问题后怎么撤销

## 建议回滚话术

- “如果改完以后命令找不到了，就把日志里的旧值恢复回去，再打开一个新终端测试。”
- “如果程序还是读取旧值，先关闭并重新打开终端或相关程序，再继续测试。”
