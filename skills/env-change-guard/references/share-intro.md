# env-change-guard 介绍文案（草稿）

## 一句话介绍

一个专门给 OpenClaw 新手准备的防呆技能：修改环境变量前先解释、改完自动记日志、还会告诉你怎么验证和回滚，尽量避免把系统越改越乱。

## 适合谁

- 刚开始用 OpenClaw 的用户
- 不太懂系统变量 / 环境变量的人
- 经常要配置 Python、Node、Java、ADB、Android SDK 的人
- 想保留变更记录，方便排查问题的人

## 解决什么问题

很多小白在配环境变量时最怕三件事：

1. 不知道自己改了什么
2. 改坏了不会回退
3. PATH 越改越乱，最后命令找不到

这个技能的作用，就是把“改环境变量”这件事变得更稳一点：

- 先说明要改什么
- 先读取当前值
- 判断风险高不高
- 改完写日志
- 告诉你怎么验证
- 告诉你怎么撤销

## 典型场景

- 给 Python / Node / Java 加 PATH
- 配置 `JAVA_HOME`
- 配置 Android SDK / ADB
- 排查 OpenClaw 找不到命令
- 清理混乱的 PATH

## 核心价值

它不是帮你“乱改系统”，而是帮你“有记录地改、能回滚地改、尽量少出错地改”。
