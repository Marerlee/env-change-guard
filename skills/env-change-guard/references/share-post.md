# env-change-guard 对外介绍文案

## 版本一：短介绍

我做了一个给 OpenClaw 新手用的小技能：`env-change-guard`。

它专门处理环境变量 / 系统变量这类容易把人搞晕的配置操作。核心思路不是“帮你猛改”，而是“先解释、再修改、全程留痕、支持回滚”。

适合这些场景：
- 配 Python / Node / Java / ADB / Android SDK
- 给 PATH 加目录
- 设置 `JAVA_HOME`
- 排查命令找不到
- 清理混乱的 PATH

这个技能会要求代理：
- 修改前先读当前值
- 明确告诉用户风险高低
- 尽量做最小改动
- 改完写日志到 `logs/env-change-log.md`
- 告诉用户怎么验证、怎么撤销

比较适合不太懂系统配置、又怕把电脑环境搞乱的人。

## 版本二：稍长一点

很多人刚开始用 OpenClaw 时，会让代理帮忙配环境变量、改 PATH、设 `JAVA_HOME`、装 Python / Node / Java 之类的东西。

问题是，这类操作看起来简单，实际很容易越改越乱：
- 不知道改的是用户变量还是系统变量
- PATH 被覆盖了
- 改完为什么命令还是不生效也不明白
- 出问题时根本不知道怎么回滚

`env-change-guard` 这个技能就是为这种情况准备的。

它不是一个“自动乱改系统”的技能，而是一个“环境变量防呆技能”。它会要求代理在每次修改环境变量前：
- 先解释要改什么
- 先读当前值
- 判断风险
- 尽量只做最小改动
- 把变更写进日志
- 最后告诉用户怎么验证和怎么撤销

默认日志位置是：`logs/env-change-log.md`

如果你经常需要帮新手配 Python、Node、Java、ADB、Android SDK，或者你自己就属于“知道要改，但又怕改乱”的类型，这个技能会很有用。
