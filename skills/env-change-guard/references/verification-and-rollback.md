# 验证与回滚速查

## 验证方法示例

- 打开一个新终端，再读取变量值
- 执行相关命令，例如：`python --version`、`java -version`、`adb version`
- 在 Windows 下用 `where 命令名` 确认命令路径
- 如果是 GUI 程序依赖的变量，先完全重启程序再测试

## 回滚方法示例

- 根据 `logs/env-change-log.md` 中记录的旧值恢复
- 如果只是 PATH 新增了一段路径，就只删除那一段，不要整体重写
- 如果改的是 shell / PowerShell 配置文件，就把改动的那一行恢复，并重新打开终端
- 如果不确定是哪次修改导致问题，优先回滚最近一次改动后再测试

## 规则

只要是中风险或高风险修改，结束前都必须同时给出：

- 至少一种验证方法
- 至少一种回滚方法
