# 环境变量变更记录技能

这个技能只做两件事：

1. 记录这次改了什么
2. 向电脑小白解释这次改动会有什么影响

## 使用要求

每次修改环境变量或系统变量时，都要做到：

- 修改前先说明准备改什么
- 修改后明确记录改了什么
- 用简单的话解释这次改动会影响什么

## 记录要求

默认把记录写到：

- `logs/env-change-log.md`

每次记录至少包含：

- 时间
- 改了哪个变量
- 修改前是什么
- 修改后是什么
- 为什么要改

## 给小白的解释方式

默认把用户当成不懂系统配置的人。

所以解释时要尽量简单，重点说清楚：

- 这次改的是哪个变量
- 它大概是干什么用的
- 改完以后哪些命令或程序可能会受影响
- 是否需要重新打开终端或程序

不要堆太多术语，不要写得像技术文档。

## 典型触发场景

- “帮我配环境变量”
- “把 Python 加到 PATH”
- “设置 JAVA_HOME”
- “配置 ADB / Android SDK”
- “改一下系统变量”
- “我不太懂，你解释简单一点”

## 完成前检查

结束前确认：

- 已记录改动内容
- 已用简单语言解释影响
