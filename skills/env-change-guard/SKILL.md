# 环境变量防呆技能

使用这个技能时，要把“修改环境变量”当成一件需要留痕、可回滚的配置操作来处理。

## 核心规则

不要静默修改环境变量。

每次都必须按这个顺序做：

1. 用简单的话说明准备改什么。
2. 先读取当前值。
3. 说明这次操作属于：
   - 低风险
   - 中风险
   - 高风险
4. 尽量只做最小改动。
5. 把变更写入 `logs/env-change-log.md`。
6. 告诉用户怎么验证是否生效。
7. 告诉用户如果出问题该怎么回滚。

## 面向新手的表达方式

默认把用户当成不熟悉系统配置的人。

所以要解释清楚：

- 这个变量是干什么的
- 为什么要改它
- 改完会影响什么
- 如果不对，怎么撤销

表达尽量短、直白、少术语。

## 风险等级

### 低风险
- 新增一个变量
- 只在当前进程临时生效的修改

### 中风险
- 修改已有的用户变量
- 往用户级 PATH 里新增一个目录

### 高风险
- 修改系统级 PATH
- 删除变量
- 直接覆盖 PATH，而不是小范围追加/调整
- 清理已经很乱的 PATH

如果是高风险操作，必须明确告诉用户“这是高风险”。

## PATH 安全规则

PATH 很容易改坏，处理时必须遵守：

- 先读取完整的当前 PATH
- 尽量一次只加一个目录
- 优先追加/前置，不要整体覆盖
- 日志里保留旧值
- 修改后提醒用户打开新终端再测试命令

## Windows 说明

在 Windows 下，必须明确告诉用户这次修改属于：

- 仅当前进程
- 当前用户级
- 整个系统级

除非确实有必要，否则优先使用**用户级**修改，不要轻易动系统级。

## 日志要求

每次修改都写到：

- `logs/env-change-log.md`

每条日志至少要包含：

- 时间
- 改了什么
- 为什么改
- 修改前的值
- 修改后的值
- 修改范围
- 验证方法
- 回滚方法

## 典型触发场景

遇到这些请求时应该触发本技能：

- “帮我配环境变量”
- “把 Python / Java / Node 加到 PATH”
- “我不懂环境变量，别把系统搞坏”
- “配置 JAVA_HOME / ADB / Android SDK”
- “清理一下 PATH，太乱了”
- “OpenClaw 找不到命令，看看是不是环境变量问题”

## 完成前检查

结束前确认：

- 已读取当前值
- 已说明风险等级
- 已做最小必要改动
- 已写日志
- 已告诉用户如何验证
- 已告诉用户如何回滚
